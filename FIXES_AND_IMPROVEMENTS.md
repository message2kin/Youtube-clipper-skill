## 問題總結

在首次測試運行中，我們遇到了以下問題：

### 1. Python 依賴缺失 ✅ 已修復

**問題**:

```
ModuleNotFoundError: No module named 'yt_dlp'
ModuleNotFoundError: No module named 'pysrt'
```

**原因**: macOS 的 Python 環境為外部管理環境（externally-managed），默認不允許使用 pip 安裝包

**解決方案**:

```bash
pip3 install --break-system-packages yt-dlp pysrt python-dotenv
```

**狀態**: ✅ 已安裝並驗證通過

---

### 2. 下載進度鈎子除零錯誤 ✅ 已修復

**問題**:

```python
TypeError: unsupported operand type(s) for /: 'int' and 'NoneType'
```

**位置**: `scripts/download_video.py` 的 `_progress_hook` 函數第 161 行

**原因**: 某些影片在下載時不提供 `total_bytes` 信息，導致除法運算失敗

**修復**:

```python
# 修復前
if 'downloaded_bytes' in d and 'total_bytes' in d:
    percent = d['downloaded_bytes'] / d['total_bytes'] * 100

# 修復後
if 'downloaded_bytes' in d and 'total_bytes' in d and d['total_bytes']:
    percent = d['downloaded_bytes'] / d['total_bytes'] * 100
```

同時添加了備用顯示邏輯，當無法獲取總大小時，只顯示已下載大小和速度

**文件**: `scripts/download_video.py:156-178`

**狀態**: ✅ 已修復並測試通過

---

### 3. 文件名特殊字符問題 ✅ 已修復

**問題**:
原始影片文件名包含特殊字符（單引號和特殊冒號）導致路徑處理困難

```
Anthropic's Amodei on AI： Power and Risk [Ckt1cj0xjRM].mp4
```

**測試時的臨時方案**: 手動重命名為 `video.mp4` 和 `subtitle.vtt`

**根本原因**:

- yt-dlp 的輸出模板使用了影片標題，標題中可能包含各種特殊字符
- 特別是某些 YouTube 影片標題中的冒號可能是全角字符（：而非:）
- 文件名中的單引號也會導致 shell 命令解析問題

**永久修復**:
修改 `scripts/download_video.py` 的輸出模板，只使用影片 ID（保證無特殊字符）

```python
# 修復前（第 70 行）
'outtmpl': str(output_dir / '%(title)s [%(id)s].%(ext)s'),

# 修復後
'outtmpl': str(output_dir / '%(id)s.%(ext)s'),
```

**優勢**:

- 影片 ID 只包含字母、數字、短橫線和下劃線（如 `Ckt1cj0xjRM`）
- 完全避免特殊字符和空格
- 文件名簡潔，兼容性好
- 影片 ID 是唯一的，不會衝突

**文件**: `scripts/download_video.py:67`

**狀態**: ✅ 已修復

---

### 4. 輸出目錄位置不符合預期 ✅ 已修復

**問題**:
測試時，輸出文件保存在 `/tmp/youtube_clipper_output/` 目錄，而不是用户當前工作目錄

**用户反饋**:
"我覺得輸出結果需要放到當前打開的文件夾裏面"

**原因**:
測試腳本使用了臨時目錄，而 `utils.py` 中的 `create_output_dir()` 默認使用 `~/Videos/youtube-clips`

**修復**:

1. **utils.py** (第 146 行):

```python
# 修復前
if base_dir is None:
    base_dir = Path.home() / "Videos" / "youtube-clips"

# 修復後
if base_dir is None:
    base_dir = Path.cwd() / "youtube-clips"
```

2. **SKILL.md 文檔更新**:

- 階段 6 輸出目錄説明: `~/Videos/youtube-clips/` → `./youtube-clips/`
- 示例輸出路徑更新
- 添加説明：輸出目錄位於當前工作目錄下

**優勢**:

- 輸出文件在用户當前工作目錄，更符合預期
- 方便管理和查找生成的文件
- 不會污染用户的 Videos 目錄

**文件**:

- `scripts/utils.py:131-148`
- `SKILL.md:240-270`

**狀態**: ✅ 已修復
